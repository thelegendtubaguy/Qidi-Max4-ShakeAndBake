## Why

Max 4 CoreXY belt-path captures need external analysis that compares A and B vibration responses without adding numerical work to Klipper. The analyzer must handle noisy LIS2DW data and invalid PSDs with explicit diagnostics.

## What Changes

- Add `shakeandbake analyze belts <capture>` for external A/B belt-path analysis.
- Read paired A and B measurement blocks from a versioned capture artifact.
- Validate samples and PSD arrays before comparison.
- Compute PSD curves, peak lists, paired and unpaired peaks, normalized area difference, amplitude ratios, and correlation only when valid.
- Produce analysis JSON, human-readable summary, and graph files.
- Report QIDI closed-loop X/Y motor metadata when available.
- Return explicit invalid-result diagnostics instead of `NaN`, `0%`, or silent success.

## Capabilities

### New Capabilities
- `external-belt-analysis`: Offline CoreXY A/B belt-path comparison for Shake&Bake capture artifacts.

### Modified Capabilities

## Impact

- Affects external analyzer CLI and analysis library code.
- Depends on capture artifacts generated by belt-path data acquisition.
- Adds fixtures for valid paired captures, missing path captures, degenerate PSDs, non-finite data, constant data, noisy LIS2DW-like data, paired peaks, and unpaired peaks.
- Does not collect printer data, run inside Klipper, edit printer configuration, or support Z-axis belt behavior.
